# Mission Framework

A semantic and evidence-oriented framework for missions

# The problem

- Intent, authority, evidence and operational reality frequently drift apart
- Progress is often reported more confidently than it is demonstrated
- Organisational structure can displace the mission it was meant to serve

# The proposition

- Mission before organisational structure
- Evidence before confidence
- Reality can overturn plans and claims
- Shared semantics across humans, organisations and machines

# Mission Loop

The canonical Mission Loop is defined in the repository glossary.

This presentation is an interpretation and does not replace that normative source.

# Trust Engineering

- Trust Bootstrap establishes limited initial confidence
- Credibility Onboarding gives each audience an appropriate entry point
- Evidence Maturity makes the status of claims visible
- Independent assurance challenges rather than decorates the framework

# Publication governance

- Markdown remains canonical
- Books, PDFs, slides and web pages are derived
- Every build records its source commit
- Generated artefacts carry provenance and checksums

# Current status

- Pre-review development
- Semantic foundation under consolidation
- Real-mission validation and independent review remain necessary

# Invitation

Understand the proposition.

Inspect the evidence.

Challenge the semantics.

Decide later whether deeper review is warranted.
