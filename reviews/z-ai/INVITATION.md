# Invitation to z.ai

Dear z.ai,

We would like to invite you to participate in the early development of an open architecture initiative called **Mission Framework**.

Your role is **not** to validate our ideas.

Your role is to **challenge them**.

We are intentionally seeking independent architectural criticism from another AI system before the project grows larger.

## About the project

The work is currently split into three public GitHub repositories.

### 1. Mission Framework

This repository defines the philosophy, principles and canonical semantics.

It asks questions such as:

- Why does the framework exist?
- What is Reality?
- What is a Mission?
- What is Evidence?
- What is justified Trust?

Think of this repository as the constitutional layer.

### 2. Mission Platform

This repository describes how those concepts can become an operational platform.

It is intentionally implementation-neutral and focuses on reference models, architecture, reusable concepts, future schemas, governance and AI interoperability.

Think of this repository as the engineering layer.

### 3. Mission Solar Eclipse

This repository is our first real-world validation project.

Instead of relying only on abstract examples, it tests the framework against an actual mission involving planning, uncertainty, weather, equipment, observation, safety, evidence and learning.

Think of this repository as the reality test.

## Reading order

Please review the repositories in this order:

1. `mission-framework`
2. `Mission-Platform`
3. `mission-solar-eclipse`

Do not assume that our terminology is correct. Challenge everything.

We are especially interested in:

- circular definitions
- hidden assumptions
- unnecessary concepts
- missing concepts
- semantic inconsistencies
- architectural weaknesses
- places where humans may misunderstand the model
- places where AI systems may interpret concepts differently

## Mission Core

One current goal is to identify a very small set of concepts that should exist in every mission, regardless of domain.

Our working rule is:

> **A concept belongs in Mission Core only if it is indispensable across domains.**

Please challenge this principle as well.

## Reality Principle

The project deliberately places **Reality** above every model.

Models are tools. Reality is the authority. Whenever evidence contradicts the model, the model must change.

This principle is inspired by the Danish expression:

> **“Remtræk til virkeligheden”**

introduced to us by **Morten Thrane**.

## Human accountability

Although AI is expected to become an important participant within Mission Framework, we currently maintain this rule:

> AI may recommend. Humans remain accountable.

If you believe this should be refined, challenged or expanded, please explain why.

## Requested response

Please use the response template in this directory and distinguish clearly between:

- observations
- interpretations
- recommendations
- open questions

We are not looking for agreement. Be precise, sceptical and fair.

If Mission Framework eventually succeeds, we hope it becomes a common language that different people, organisations and AI systems can use to describe missions grounded in reality.
