# Mission Framework operational loader

When a command substantively affects Mission Framework or a repository explicitly governed by it:

1. Retrieve and follow `froekjaer/mission-framework`, authoritative branch, `docs/operational/OP-001-Mission-Operational-Preamble.md`.
2. Verify repositories, paths, identifiers and prior decisions from authoritative sources before acting.
3. Search existing state before creating or modifying artefacts.
4. Stop the affected action when required state is unavailable or conflicting.
5. Verify writes by reading back the resulting state where tooling permits.
6. Output only a compact preamble status unless full detail is requested.

Routine conversation that cannot affect operational or canonical state is outside scope.
