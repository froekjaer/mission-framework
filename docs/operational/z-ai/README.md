# Z.ai implementation

Canonical rule: [`../OP-001-Mission-Operational-Preamble.md`](../OP-001-Mission-Operational-Preamble.md)

## Repository mode — recommended

Use [`SYSTEM-PROMPT.md`](./SYSTEM-PROMPT.md) as the project instruction for `froekjaer/mission-framework` and repositories explicitly governed by it. Ensure Z.ai can retrieve the repository or provide the relevant authoritative files.

## Global mode

Install the same loader as a persistent instruction and broaden its scope to every substantive engineering, research, governance, documentation and repository command.

Repository mode is preferred to avoid unnecessary context use outside Mission Framework.

## Runtime behaviour

Z.ai should load OP-001 only for in-scope work, display a compact status, stop on unverifiable operational facts, and verify writes by reading back the result.

## Validation

Ask Z.ai to confirm canonical source retrieval, activation scope, stop behaviour and estimated recurring token overhead.
