# Foundation v1.1 Release Plan

**Status:** Release candidate plan  
**Release theme:** From foundation to validation

## Release intent

Foundation v1.1 does not claim that Mission Framework has been empirically validated.

It establishes the rules, evidence structure and governance needed to decide whether the framework deserves continued existence and expansion.

## Sprint 1 — Preserve review evidence

Repository: `froekjaer/collaborative-intelligence`

- preserve all original reviewer artefacts unchanged
- preserve reviewer metadata and reviewed commit identifiers
- publish review index, status and timeline
- keep synthesis separate from original evidence
- record missing or incomplete artefacts without reconstruction or invention

## Sprint 2 — Baseline synthesis

Repository: `froekjaer/collaborative-intelligence`

- preserve the Baseline v0.1 synthesis
- maintain a consensus matrix and unique findings
- distinguish evidence, interpretation and recommendation
- publish an implementation backlog with traceability to review findings
- identify unresolved conflicts rather than hiding them

## Sprint 3 — Foundation v1.1 normative update

Repository: `froekjaer/mission-framework`

Required documents:

- Foundation v1.1 Validation Update
- Validation and Scientific Method
- Existence and Value Test
- Framework Stabilisation Criteria
- Baseline Manifest
- updated README and foundational-document index

## Sprint 4 — Governance and stabilisation

Repositories: programme-wide

- establish minimum conformance roadmap
- define baseline and release manifests
- create an ADR or equivalent decision-record convention
- create the Mission Solar Eclipse validation plan
- identify release blockers, warnings and deferred work

## Release blockers

Foundation v1.1 shall not be tagged while any of the following remain unresolved:

- an original completed review is known but not preserved or linked
- the synthesis is presented as a substitute for source reviews
- the validation method lacks a credible comparison requirement
- the existence criterion cannot produce a negative conclusion
- the baseline manifest contains guessed commit identifiers
- Mission Solar Eclipse has no bounded validation proposition
- release documents contradict canonical semantics without a Framework Finding

## Release warnings

The following may remain with an explicit disposition:

- incomplete automation
- limited external participation
- only one initial reference mission
- provisional measures requiring refinement
- unresolved questions that do not prevent the first test

## Deferred beyond v1.1

The following are not required for release:

- a complete Mission Platform
- broad ontology formalisation
- multiple validated reference missions
- universal conformance certification
- claims of general scientific validity
- large-scale external adoption

## Release evidence

The release record shall contain:

- exact commit identifiers
- pull-request review history
- review archive status
- synthesis status
- validation plan status
- known limitations
- accountable release decision

## Proposed release decision language

> Foundation v1.1 is released as a validation-ready semantic and governance baseline. It is not released as proof that Mission Framework improves mission outcomes. That proposition remains under empirical test.

## After release

The next programme milestone is not Foundation v1.2 by default.

The next milestone is the first completed, published and independently challenged reference-mission validation result.
