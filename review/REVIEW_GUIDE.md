# Review Guide

Thank you for considering a review of Mission Framework v0.2.

The review is intended to test whether the proposed concepts, methods and governance structures are understandable, defensible and useful. Reviewers are not asked to endorse the project.

## Core questions

1. Which ideas appear useful or original?
2. Which ideas already exist under better-established terminology?
3. Which claims require stronger evidence or qualification?
4. Which assumptions are hidden?
5. Which definitions are unclear, circular or unnecessary?
6. What would falsify the framework's central propositions?
7. Where would the approach fail in practice?
8. Which parts should be removed rather than improved?

## Review principles

Please distinguish, where possible, between:

- factual error
- missing evidence
- conceptual disagreement
- unclear language
- practical limitation
- normative disagreement
- implementation concern

Strong criticism is welcome. Specific examples, counterexamples, literature references and alternative formulations are especially useful.

## Scope

A review may cover the entire repository or a bounded topic. Reviewers should state the material reviewed, their perspective and any relevant limitations or conflicts of interest.

## Submitting feedback

Feedback may be submitted through a GitHub issue, pull request or the review template in this directory.

Participation should not be interpreted as endorsement of Mission Framework, its contributors or its conclusions.
